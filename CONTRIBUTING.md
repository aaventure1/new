# Contributing to AAVenture

Thanks for helping build AAVenture.

## Fast Start
1. Fork the repository and clone your fork.
2. Create a feature branch from `main`.
3. Run locally:
   - `npm install`
   - `npm run dev`
4. Open a pull request with a clear summary.

## Where Help Is Needed
- Bug fixes in auth/chat/meetings flows
- UI polish and accessibility improvements
- Tests and reliability improvements
- Docs and onboarding improvements

## Pull Request Rules
- Keep PRs small and focused.
- Include steps to test your change.
- Do not commit secrets or private keys.
- Update docs when behavior changes.

## Good First Contributions
- Fix typo/documentation issues
- Improve error messages and empty states
- Add tests around existing API routes
- Improve mobile responsiveness

## Need Direction?
Open an issue with label `help wanted` and describe what you want to work on.
