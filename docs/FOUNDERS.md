# Founders Operating Checklist

Use this file as the shared baseline before inviting long-term collaborators.

## 1) Company and Ownership
- [ ] Company entity formed (LLC or equivalent).
- [ ] Company bank account created.
- [ ] Cap table started and versioned.
- [ ] Founder equity split documented in writing.
- [ ] Vesting terms documented (common default: 4 years, 1-year cliff).

## 2) Legal and IP Protection
- [ ] Founders signed IP assignment to company.
- [ ] Confidentiality + invention assignment signed.
- [ ] If using contractors, contractor IP assignment template ready.
- [ ] Trademark/domain ownership moved under company accounts where possible.

## 3) Repo and Access Control
- [ ] Primary repo owned by org, not personal account.
- [ ] Minimum two admins (founders only).
- [ ] Everyone else has least-privilege access.
- [ ] `main` branch is protected.
- [ ] Two-factor authentication required for org members.

## 4) Engineering Workflow
- [ ] PRs required for `main`.
- [ ] At least 1 approval required on each PR.
- [ ] Force-push blocked on `main`.
- [ ] CI checks identified and required before merge.
- [ ] Security-sensitive changes require founder review.

## 5) Finance and Ops Basics
- [ ] Budget owner assigned.
- [ ] Monthly spend tracking in place.
- [ ] Vendor accounts use company billing, not personal cards where possible.
- [ ] Incident contact path defined (who handles production issues first).

## 6) Founder Alignment (Write This Down)
- [ ] Roles and decision ownership by area (product, tech, growth, finance).
- [ ] Weekly founder sync on calendar.
- [ ] Conflict resolution rule written down (who breaks ties and how).
- [ ] Definition of done for the next 90 days agreed.

## Suggested Defaults (Edit as Needed)
- Equity vesting: 4 years with 1-year cliff
- Board/decision cadence: weekly founder review
- Merge policy: PR + 1 approval + passing checks
- Emergency production changes: documented in PR after incident

## Notes
Keep this file updated as decisions become final. If legal terms change, add a dated note with what changed and why.
