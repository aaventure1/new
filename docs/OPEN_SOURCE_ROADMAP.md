# Open Source Roadmap

This roadmap is focused on attracting contributors while improving core product quality.

## Phase 1: Contributor Onboarding
- [ ] Tighten setup reliability on macOS/Linux
- [ ] Add API smoke tests for auth/meetings
- [ ] Improve error messages and fallback states

## Phase 2: Stability
- [ ] Add request validation across all API routes
- [ ] Add rate-limit coverage for sensitive endpoints
- [ ] Improve server logging structure

## Phase 3: User Experience
- [ ] Mobile-first UI polish for chat and meetings
- [ ] Improve accessibility (keyboard navigation, ARIA, contrast)
- [ ] Add guided first-time user flow

## Phase 4: Growth
- [ ] Public demo environment with seeded data
- [ ] Contributor showcase in README
- [ ] Monthly community issue triage
